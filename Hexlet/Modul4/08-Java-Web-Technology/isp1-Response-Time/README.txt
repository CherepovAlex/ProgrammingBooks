В современных веб-приложениях производительность и отзывчивость являются важными аспектами,
влияющими на пользовательский опыт. Одним из показателей производительности является время выполнения запросов,
которое определяет, насколько быстро сервер обрабатывает входящие запросы и возвращает ответы клиентам.
Измерение времени выполнения запросов позволяет разработчикам выявлять узкие места в производительности,
оптимизировать обработку данных и улучшать общую эффективность приложения.

В этой задаче мы сосредоточимся на реализации механизма, который будет отслеживать и измерять время выполнения
HTTP-запросов в приложении

src/main/java/App.java
Создайте и подключите мидлвару, которая будет считать время выполнения запроса в приложении и устанавливать
его в качестве значения в кастомный заголовок X-Processing-Time ответа

Соберите приложение командой make build и откройте веб доступ. По адресу /products приложение отдает
список всех продуктов в базе данных. При помощи параметра строки запроса limit можно ограничить
количество отдаваемых товаров. Делайте разные запросы к приложению и посмотрите, как меняется время выполнения.
 Заголовки ответа можно посмотреть прямо в браузере в инструментах разработчика. Второй вариант —
 скопировать адрес из веб-доступа и отправлять к нему запросы при помощи Curl, Postman или любой другой утилиты

Подсказки
Вы можете добавить в контекст любую информацию и прочитать ее из контекста при помощи перегруженного метода attribute()
